Why Python?

  • Rich ML Ecosystem: NumPy, Pandas, PyTorch, TensorFlow, Scikit-learn all native
  • Rapid Prototyping: Fast development cycle perfect for academic projects
  • OpenCV Integration: cv2 provides seamless video processing
  • Community Support: Extensive documentation and examples

Why YOLOv8?

  • State-of-the-Art: Latest YOLO version with best speed-accuracy trade-off
  • Real-time Performance: 30+ FPS on consumer hardware
  • Pre-trained Models: COCO dataset with 80 object classes
  • Easy Fine-tuning: Can be trained on custom datasets

Why PyTorch?

  • Dynamic Graphs: Easier debugging and model experimentation
  • GPU Acceleration: CUDA support for faster inference
  • Pre-trained Models: torchvision hub with FaceNet, ResNet, etc.
  • Industry Standard: Used by Facebook AI, Tesla, and OpenAI

Why Flask?

  • Lightweight: Minimal overhead for video streaming
  • MJPEG Streaming: Native support for real-time video feeds
  • REST API: Easy to create endpoints for frontend
  • Python Native: Seamless integration with ML code
Architecture

System Architecture

Modular, scalable architecture designed for extensibility and real-time performance.

Input Layer

  • Video Stream Module: Multi-source video capture (webcam, IP cameras, RTSP)
  • Threaded Capture: Async frame buffering for smooth processing
  • Preprocessing: Resolution scaling, normalization, format conversion

Processing Layer

  • Object Detection: YOLOv8 with GPU acceleration
  • Tracking: Deep SORT multi-object tracker
  • Recognition: FaceNet embeddings + cosine similarity
  • Analysis: Behavior patterns, anomaly detection

Output Layer

  • Dashboard: Real-time web interface with glassmorphism UI
  • Alerts: Priority-based multi-channel notifications
  • Storage: SQLite for alerts, JSON for analytics
  • Export: CSV, JSON, PDF report generation

Performance Optimizations

Frame Skipping

Process detection every 3rd frame while displaying all frames. Caches results for skipped frames to maintain visual continuity.

Threaded Processing

Separate threads for video capture, detection, and display. Eliminates I/O blocking with queue-based frame buffering.

GPU Acceleration

CUDA-enabled inference for PyTorch models. Auto-detects GPU availability with graceful CPU fallback.

Why This Project?

The motivation, challenges, and learning outcomes from building this system.

01
Real-World Problem

Security is a universal concern. Traditional CCTV systems are passive - they record but don't analyze. Our system makes surveillance proactive.

02
AI/ML Application

03
Industry Relevance

Smart cities, retail analytics, and enterprise security are multi-billion dollar industries actively hiring for these skills.

04
Technical Depth

Covers system design, real-time streaming, database management, web development, and ML model deployment - full-stack AI experience.

Challenges Faced

  • Real-time Processing: Balancing accuracy vs. speed on limited hardware
  • False Positives: Reducing alert fatigue from over-sensitive detection
  • Model Integration: Combining multiple ML models efficiently
  • Memory Management: Handling video streams without memory leaks

Solutions Implemented

  • Frame Skipping: Process every 3rd frame, cache results for others
  • Alert Throttling: 30-second cooldown per alert type
  • Modular Architecture: Separate feature classes with common interfaces
  • Threaded Processing: Queue-based buffering with daemon threads
